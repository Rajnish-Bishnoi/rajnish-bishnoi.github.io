3-Tier AWS Architecture

Application Load Balancer • Auto Scaling • RDS

Overview

This project demonstrates a **3-tier architecture** on AWS, using an Application Load Balancer for traffic distribution, an Auto Scaling group for the application layer, and Amazon RDS for the database layer.

Architecture Diagram

Key Components

  • Application Load Balancer (ALB): Distributes incoming traffic across multiple EC2 instances.
  • Auto Scaling Group: Automatically adjusts the number of EC2 instances based on demand.
  • Amazon RDS: Managed relational database service for persistent data storage.

Benefits

  • High availability
  • Scalability on demand
  • Fault tolerance

Deployment Steps

  1. Create VPC, subnets, and security groups.
  2. Deploy ALB in public subnets.
  3. Configure Auto Scaling group in private subnets.
  4. Launch RDS instance in database subnets.
  5. Test application access through ALB endpoint.
